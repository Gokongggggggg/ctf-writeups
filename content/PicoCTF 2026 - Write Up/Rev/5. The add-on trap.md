>[!note]- Explanation
> - After extracting, we inspect the contents of `extracted_addon/`
> - The most important file here is **`manifest.json`**
>
> - Why?
>   - In browser extensions, `manifest.json` acts like an **entry point**
>   - Similar to `main()` in C, but instead of code execution:
>     - it defines the **configuration and behavior** of the extension
>
> - It usually contains:
>   - which scripts run
>   - what pages are loaded (e.g., `popup.html`)
>   - permissions and resources
>
> - So our next step is to read `manifest.json` to understand how the extension works

## Decryption

>[!note]- Explanation
> - The ciphertext starts with `gAAAA...`, which is a strong indicator of **Fernet encryption**
>
> - Fernet is:
>   - a symmetric encryption scheme from Python (`cryptography` library)
>   - uses a **URL-safe base64 key (32 bytes)**
>   - provides authenticated encryption (AES + HMAC)
>
> - This matches perfectly with:
>   - the format of `webhookUrl`
>   - the comment about a **32 url-safe base64-encoded key**
>
> - So we can decrypt it using the provided key:
>   ```python
>   from cryptography.fernet import Fernet
>
>   key = b"PASTE_KEY_HERE"
>   ciphertext = b"PASTE_WEBHOOK_URL_HERE"
>
>   f = Fernet(key)
>   print(f.decrypt(ciphertext).decode())
>   ```
