# Observation

- `packets.pcap` is a **packet capture** file  
  - it stores recorded network traffic
  - in this case, likely the captured **file transfer**

- We also get `encrypt.py`
  - this shows how the file was processed **before being sent**
  - so it gives us the encoding/encryption logic used during transfer

## Solver.py

``` python
from scapy.all import rdpcap, Raw

packets = rdpcap('packets.pcap')
sessions = packets.sessions()

for session_id, session_packets in sessions.items():
    encrypted_payload = b""
    for pkt in session_packets:
        if pkt.haslayer(Raw):
            encrypted_payload += pkt[Raw].load

    if not encrypted_payload:
        continue

    decrypted = bytearray((b - 42) % 256 for b in encrypted_payload)

    if decrypted.startswith(b'\xff\xd8\xff'):
        ext = '.jpg'
    elif decrypted.startswith(b'\x89PNG'):
        ext = '.png'
    else:
        try:
            decrypted.decode('utf-8')
            ext = '.txt'
        except:
            ext = '.bin'

    filename = f"extracted_{session_id.replace(':', '_')}{ext}"
    with open(filename, "wb") as f:
        f.write(decrypted)
    

```

>[!note]- Explanation
> - After rebuilding each raw payload from `packets.pcap`, we decrypt it with:
>   ```python
>   (b - 42) % 256
>   ```
>
> - This comes directly from `encrypt.py`, where each byte was encoded as:
>   ```python
>   (b + 42) % 256
>   ```
> - So the reverse is simply:
>   - encryption: `+42 mod 256`
>   - decryption: `-42 mod 256`
>
> - Example:
>   - `4 + 2 mod 10 = 6`
>   - then `6 - 2 mod 10 = 4`
>
>
>
> - At first, I tried treating the recovered data as **text**, but it came out as gibberish
> - That suggested the payload was probably **not plain text**
> - So the next intuition was to check common file signatures like **JPEG** or **PNG**
>
> - That is why the solver checks **magic bytes**
>   - Magic bytes are the first bytes of a file that identify its format
>
> - In this script:
>   - `b'\xff\xd8\xff'` → JPEG
>   - `b'\x89PNG'` → PNG
>   - otherwise:
>     - if it decodes as UTF-8 → save as `.txt`
>     - else → save as `.bin`
>
> - So the overall logic is:
>   - rebuild payload from packets
>   - undo the byte shift with `-42`
>   - identify the file type from its magic bytes
