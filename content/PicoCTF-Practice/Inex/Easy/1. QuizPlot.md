>  
> Because of this, the program will not throw an error and will continue writing beyond the buffer, leading to an overflow.

> NX (Non-Executable) prevents execution of code on the stack, meaning we cannot directly run shellcode placed in the buffer.  
>  
> To bypass this, we use **ROP (Return-Oriented Programming)**, which works by reusing existing code in the binary.  
>  
> Instead of executing code from the stack, ROP chains together small instruction sequences (gadgets) that already exist in executable memory.  
>  
> This is typically done by overwriting the return address to point to another function (such as `win()`), or chaining multiple gadgets to achieve arbitrary behavior.
