>[!note]- Observation
> - At first, I thought the value shown at the top was some kind of **seed**
> - My initial idea was:
>   - reverse the binary
>   - use that seed to recover the secret
>
> - But that interpretation turned out to be wrong
>
> - The key clue is the wording:
>   - **“Here’s the next binary in bytes:”**
>
> - That means the binary dump belongs to the **next challenge step**
> - So it would make no sense if the value printed before it was only a seed for reversing that same binary
>
> - After looking closer, the value printed before the dump is actually the **secret itself**
> - The hex binary is just there to mislead us into overthinking it
>
> ### Conclusion
> - Read the leaked secret from the top line
> - Send it back
> - Repeat for all 20 rounds

## Solve.py

``` python
from pwn import *

r = remote('mysterious-sea.picoctf.net', 63753)

r.recvuntil(b"Good luck >:)\n")

for i in range(20):
    print(f"\n=== Round {i+1}/20 ===")

    secret = r.recvline().strip()
    print(f"[*] Secret: {secret}")

    r.recvline()  # "Here's the next binary in bytes:"

    hex_data = r.recvuntil(b"What's the secret?:").strip()

    print(f"[*] Hex length: {len(hex_data)} chars = {len(hex_data)//2} bytes")

    r.sendline(secret)

    resp = r.recvline().strip()
    print(f"[*] Response: {resp}")

r.interactive()
```

>[!note]- Explanation
> - After connecting, we wait until `"Good luck"` appears  
> - Then for each round:
>   - read the **first line** → this already contains the secret  
>   - skip the binary dump  
>   - wait until prompt `"What's the secret?"`  
>   - send the extracted secret with `sendline()`  
>
> - Repeat this process **20 times** to complete the challenge
