## Generate_math_question ()

![[Pasted image 20260319104007.png]]

>[!note]- Explanation
> - Even though it uses `rand()`, the operation will always fall into **1 of 3 cases**:
>   - `0x2b` → `+`
>   - `0x2d` → `-`
>   - `0x2a` → `*`
>
> - So the question is always just basic math:
>   - addition
>   - subtraction
>   - multiplication
>
> - We can still solve it easily because in `main()` the program prints:
>   ```c
>   printf("What is %d %c %d? ", local_28, local_29, local_24);
>   ```
>   - `local_28` = first number
>   - `local_29` = operator
>   - `local_24` = second number
>
> ### Case visualization
> - `+` case:
>   - operator = `0x2b`
>   - result = `param_3 + param_2`
>
> - `-` case:
>   - operator = `0x2d`
>   - if needed, values are swapped first so subtraction stays non-negative
>   - result = `param_2 - param_3`
>
> - `*` case:
>   - operator = `0x2a`
>   - result = `param_3 * param_2`
>
> ### Key point
> - We do **not** need to predict `rand()`
> - The program already prints the generated numbers and operator for us
> - So we can just read the prompt and compute the answer directly

-  Solving the math only gets us past the check  
-  After that, the flag is still passed into `encode_flag(...)`  
- So to recover the real flag, we now need to analyze **`encode_flag()`** and understand how it transforms the flag

![[Pasted image 20260319105145.png]]

>[!note]- Explanation
> ### Parameters
> - `param_1` → pointer to **flag buffer** (result of reading `flag.txt`)  
> - `param_2` → **correct math answer** from `generate_math_question()`  
>
> ### Function Logic
> - Loop each character in the flag:
>   ```c
>   output[i] = flag[i] * param_2
>   ```
> - Then print all values as decimal numbers  
>
> ### Decode
> - Since encoding is multiplication:
>   - `encoded = flag[i] * answer`
> - To recover:
>   - `flag[i] = encoded / answer`
>
> - Just:
>   - take each printed number  
>   - divide by the math answer  
>   - convert to ASCII → get original flag
